# Инструкция по развертыванию на сервере (Wine Mascot Portal)

Портал разработан как **Full-Stack веб-приложение** с использованием **Express** (бэкенд) и **React + Vite** (фронтенд) с поддержкой встроенного файлового хранилища `db.json` для персистентности данных (без необходимости настраивать сложные SQL базы данных).

---

## 1. Требования к окружению

Для запуска приложения на вашем VPS/сервере вам понадобятся:
- **Node.js** версии `18.x` или выше.
- **npm** (поставляется вместе с Node.js).
- Диспетчер процессов **PM2** (рекомендуется для удержания сервера онлайн 24/7).

---

## 2. Подготовка и сборка проекта

1. **Клонируйте или загрузите проект** в рабочую папку на вашем сервере (например, `/var/www/wine-portal`).
2. **Установите все зависимости**:
   ```bash
   npm install
   ```
3. **Настройте переменные окружения**:
   Создайте файл `.env` в корневом каталоге и скопируйте туда настройки из `.env.example`:
   ```env
   # Ссылка, на которой крутится приложение (например, ваш домен)
   APP_URL="https://my-wine-portal.ru"

   # Опционально: Токен вашего Telegram бота для уведомлений администраторов
   TELEGRAM_BOT_TOKEN="582910398:AAH_XYZ...YourToken"
   ```
4. **Соберите проект для продакшена**:
   Этот шаг скомпилирует фронтенд (React в статические файлы в папке `dist`) и соберет бэкенд (`server.ts`) в единый оптимизированный CommonJS файл `dist/server.cjs` с помощью `esbuild`:
   ```bash
   npm run build
   ```

---

## 3. Запуск сервера

Вы можете запустить приложение напрямую с помощью Node.js:
```bash
NODE_ENV=production npm start
```
Приложение начнет слушать порт **3000** и автоматически обслуживать как API-маршруты, так и статические файлы фронтенда!

---

## 4. Рекомендуемый запуск через PM2 (для VPS)

Чтобы приложение работало в фоновом режиме и автоматически перезапускалось при сбоях:

1. Установите PM2 глобально:
   ```bash
   npm install -g pm2
   ```
2. Запустите проект через PM2:
   ```bash
   NODE_ENV=production pm2 start dist/server.cjs --name "wine-portal"
   ```
3. Настройте сохранение списка процессов, чтобы они запускались при перезагрузке самого сервера VPS:
   ```bash
   pm2 save
   pm2 startup
   ```

---

## 5. Настройка Telegram-бота для уведомлений

Чтобы тейки и анкеты поступали в Telegram соответствующим админам:
1. Создайте нового бота через [@BotFather](https://t.me/BotFather) и скопируйте полученный API Token.
2. Пропишите этот токен в переменную `TELEGRAM_BOT_TOKEN` вашего `.env` файла.
3. В админ-панели на сайте (`/admin-panel`), перейдите во вкладку редактирования профиля или управления администраторами.
4. Каждому администратору (включая Владельца) пропишите их персональный **Telegram User ID** в поле "Telegram ID" (узнать его можно через ботов, например, [@userinfobot](https://t.me/userinfobot)).
5. Напишите вашему созданному боту `/start` в Telegram, чтобы открыть диалог.
6. Теперь при отправке любого тейка, идеи или анкеты, бот мгновенно пришлет красивое уведомление администратору на его Telegram!
